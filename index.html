Editing & Imputation

What is editing?

Detection and correction of errors in the data.

Editing and imputing are both methods of data processing. Editing refers to the detection and correction of errors in the data.

What is imputation?

A method of correcting errors and estimating and filling in missing values.

Imputation is a method of correcting errors and estimating and filling in missing values in a data set. Where there are errors in the data set, and when these are considered to add no value in the correction process, these values are set to missing and are imputed with a plausible estimate. Alternatively, missing values may already exist in the data, and imputation may be carried out to produce a complete data set for analysis

Different types of imputation

interactive treatment
deductive imputation
model based imputation
donor based imputation

Methods of variable treatment can be grouped into one of the following categories:
interactive treatment
Deductive imputation uses logic or an understanding about the relationship between variables and units to fill in missing values. Examples include deriving a value as a function of other values, adopting a value from a related unit, and adopting a value from an earlier time point. Generally, this method is used when the true value can be derived with certainty or with a very high probability.
model based imputation
donor based imputation
Donor based imputation adopt values from an observed unit, which are then used for the missing unit. For each recipient with a missing value for variable y, a donor is identified that is similar to the recipient with respect to certain background characteristics (often referred to as matching variables) that are related to the target variable y. These methods are ideal when there are several related missing values in one record, and if the intention is to preserve the relationship between variables.

Conceptualising editing and imputation

review
select
treat

An imputation process of a data set can be broken down into the following three phases:
Review, whereby data is examined for potential problems; identifying instances of missingness and erroneous values
Select, whereby data is identified for further treatment. Of the potential problems identified in the review phase, a method is applied to determine which of these erroneous or missing cases need to be treated
Treat, whereby changes are applied to the data identified in the select phase by either correcting errors/ filling in missing values

Goals of statistical data editing

data in each record should be made to satisfy all edits by changing the fewest possible variables
imputation rules should derive automatically from edit rules
when imputation is necessary, it should maintain the joint distribution of variables

A couple of researchers, proposed a set of goals for statistical data editing. That is, when we are transforming our dataset with edit rules and imputation, what should we strive for. The Felligi-Holt principles advocate for the following:
that we create units that pass edit rules whilst making the fewest possible changes to their original values
where we use deductive imputation, it is based on our edit rules
when we have imputed the data, we've maintained the relationship between variables

CANCEIS

What is CANCEIS?

CANadian Census Edit and Imputation System

CANCEIS is a tool that is used to carry out editing and imputation. It was designed and built by Statistics Canada.
It was designed based on the Felligi-Holt principles
It was built using the C# programming language.
CANCEIS carries out deductive and donor imputation.

How does it work?

input
process
output

At a high level, you can break the CANCEIS process into:
input
process
output
At the input stage, you specify the edit rules, which tell CANCEIS how to spot invalid or implausible responses for a given variable or how to spot an incorrect or implausible combination of responses across multiple variables.
At the input stage, you also instruct CANCIES on how to find donors. Remember that with donor imputation, we are trying to impute values using respondents that we think would have responded similarly. So we give CANCEIS instructions on how to find this "similar" unit.
At the process stage:
1) units get sorted based on a set of edit rules, where they either pass and become donors or fail and become recipients
2) imputation is carried out. this can be done deductively using the edit rules. Or donors and recipients are matched on qualitative and quantitative variables, to identify nearest neighbours
3) imputed responses are evaluated against the edit rules. If they pass these edit rules, they are retained. If they fail, we repeat the search for a donor.
At the output stage, you get your edited and imputed dataset, as well as a bunch of metadata, which describes the output. And you also get a bunch of paradata, which describes the process.

CANCEIS inputs

data dicionary
decision logic table
input dataset

the data dictionary tells CANCEIS what is considered to be a valid response for a given variable.
the decision logic table, is where we specify the edit rules. This tells CANCEIS what would be an inconsistent response between two or more variables.
the input dataset has your units and items. We've got to code the missing values so CANCEIS knows when missingness needs to be imputed.

CANCEIS process


Review observations in the input dataset
Select values using the data dictionary and decision logic table
Treat data using either deductive or donor imputation ( minimum change nearest neighbour)

If we are to look at the CANCEIS methodology through the GSDEM lense, it would take the following form:
CANCIES uses the data dictionary and decision logic table to review the variables you've asked it to edit and impute; to identify potentially problematic values
Using your parameters in the data dictionary and decision logic table, it selects units for treatment
During the treatment phase, it will transform your dataset using deductive and donor imputation. For the latter, it can use either a deterministic or stochastic mechanism. That is, when it is selecting a donor it can either randomly select a donor or select the "best" possible donor.
The type of donor imputation it carries out is called minimum change nearest neighbour imputation. What this means is that when it is looking for a donor; it looks both at whether the imputed value would pass the edit rules and how far this new value is from the original value. For example if a unit's net income is reported as being greater than their gross income, this unit will be selected for treatment. During the donor search process, it will look for imputed values for both gross and net income. And then select an imputation action that would preserve the net-gross relationship but also result in smallest change to reported values.
